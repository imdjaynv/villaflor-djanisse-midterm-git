# Midterm Examination Reflection

### 1. What is Git used for?
Git is a local version control system used to track changes in source code during software development. It allows developers to save historical versions of their project, revert to previous states if mistakes occur, and manage code modifications efficiently. 

### 2. What is the difference between Git and GitHub?
Git is the actual command-line tool installed on your computer that tracks file changes locally. In contrast, GitHub is a cloud-based hosting service that stores those Git repositories online so developers can collaborate and share code. 

### 3. Why is branching useful?
Branching allows developers to isolate their work from the main codebase to experiment with new features or fix bugs safely. It ensures that the primary production code remains stable and functional while multiple team members work on different tasks simultaneously.

### 4. What command did you use to upload your local commits to GitHub?
To upload local commits to GitHub, the command `git push` is used. Typically, this is executed as `git push origin main` or `git push origin master` to send the local updates to the corresponding remote branch.
